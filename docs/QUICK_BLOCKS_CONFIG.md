## Формат `config/quick_blocks.yaml`

Файл `config/quick_blocks.yaml` описывает все группы и блоки для панели Quick Blocks. Пример рабочего файла:

```yaml
version: 1
groups:
  - id: cameras            # Уникальный идентификатор группы (строка)
    name: "Камеры"         # Отображаемое имя
    collapsed: false        # true, если группа свернута при загрузке
    order: 0                # Порядок отображения группы (0 = сверху)
    blocks:
      - id: cam_11          # Уникальный идентификатор блока
        title: "CAM 11"     # Название в UI
        command_on: "port set 1 1"   # Команда ON
        command_off: "port clear 1 1" # Команда OFF
        send_to_combo: true           # Отправлять на оба CPU при combo
        order: 0                      # Позиция внутри группы
        mode: dual                    # "dual" или "single" (без OFF)
        icon: null                    # Резерв для будущих иконок
        port: cpu1                    # cpu1, cpu2, tlm или combo
        hotkey: null                  # Горячая клавиша (опционально)
        group_id: cameras             # ID группы (дублирует связь)
```

### Поля группы

| Поле      | Тип      | Обязательное | Описание |
|-----------|----------|--------------|----------|
| `id`      | string   | да           | Уникальный ключ группы (используется как `group_id` у блоков).
| `name`    | string   | да           | Название группы в UI.
| `collapsed` | boolean | нет (default: false) | Состояние свернутости при старте.
| `order`   | integer  | нет (default: 0) | Порядок сортировки групп (0 сверху).
| `blocks`  | list     | да           | Список блоков.

### Поля блока

| Поле          | Тип      | Обязательное | Описание |
|---------------|----------|--------------|----------|
| `id`          | string   | да           | Уникальный идентификатор блока.
| `title`       | string   | да           | Заголовок кнопки в UI.
| `command_on`  | string   | да           | Команда, отправляемая при нажатии ON.
| `command_off` | string   | зависит от `mode` | Требуется, если `mode != "single"`.
| `send_to_combo` | boolean | нет (default: true) | Отправлять ли команду на оба порта при combo.
| `order`       | integer  | нет (default: 0) | Порядок элементов в группе.
| `mode`        | string   | нет (default: "dual") | `dual` — ON/OFF, `single` — только ON.
| `icon`        | string  | нет | Путь к иконке (опционально, в текущей версии не используется).
| `port`        | string   | да           | Какой порт использовать по умолчанию (`cpu1`, `cpu2`, `tlm`, `combo`).
| `hotkey`      | string|null | нет | Горячая клавиша (формат Qt, например `Ctrl+Alt+1`).
| `group_id`    | string   | да           | ID группы, к которой относится блок (должен совпадать с `groups[].id`).

### Рекомендации по заполнению

1. **Версия** (`version`): оставляйте текущую версию схемы (на момент 2026-02-25 = 1). При изменении структуры файла увеличивайте версию для удобства миграций.
2. **Порядок**: чтобы блоки отображались в нужной последовательности, используйте целые числа без пропусков (`0, 1, 2 ...`). При вставке нового блока задавайте order вручную или пересортируйте через UI.
3. **Состояния OFF**: если блок должен иметь только кнопку ON, установите `mode: single` и удалите поле `command_off`.
4. **Порты**: поле `port` определяет основной порт; если `send_to_combo = true`, команда отправляется на оба CPU (комбо режим) в дополнение к основному порту.
5. **Горячие клавиши**: при необходимости добавляйте `hotkey` в формате Qt (`Ctrl+Shift+1`). Это поле пока не используется в UI, но сохранится в конфиге.
6. **Иконки**: `icon` зарезервировано — укажите путь относительно `assets/icons`, если позже добавите визуальные индикаторы.

### Проверка файла

Перед запуском приложения убедитесь, что YAML корректен:

```bash
python -c "import yaml; yaml.safe_load(open('config/quick_blocks.yaml', 'r', encoding='utf-8'))"
```

При ошибке форматирования приложение перейдёт в fallback-конфигурацию, поэтому валидируйте файл после редактирования.

### Частые проблемы

1. **Дублирующийся `id`** — должен быть уникальным как для групп, так и блоков.
2. **Несогласованный `group_id`** — если блок ссылается на группу, которой нет в массиве `groups`, блок будет проигнорирован.
3. **Пустые команды** — `command_on` и (для dual) `command_off` должны быть строками хотя бы из одного символа.
4. **Порядок без переиндексации** — после удаления элементов перезаписывайте `order`, чтобы избежать неожиданных сортировок.

### Пример минимальной конфигурации

```yaml
version: 1
groups:
  - id: bb_group
    name: "BeagleBone"
    collapsed: false
    order: 0
    blocks:
      - id: bb_a
        title: "BB A"
        command_on: "bb a on"
        command_off: "bb a off"
        send_to_combo: true
        order: 0
        mode: dual
        icon: null
        port: combo
        hotkey: null
        group_id: bb_group
```

Сохраните файл в UTF-8 без BOM. При необходимости используйте UI-редактор Quick Blocks для операций CRUD — он автоматически поддерживает порядок и структуру файла.
